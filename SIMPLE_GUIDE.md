# Simple Guide

This project is a step-by-step workflow for turning a research question into a short experiment planning brief.

You do not need to understand agent design to use the current scaffold.

Each workflow run is saved in its own folder inside `projects/`.

## What the workflow does

It tries to:

1. understand your question,
2. organize the work into steps,
3. pause when a shortlist needs your approval,
4. and produce a short final brief.

Right now, the project has the workflow structure in place, but it does not yet connect to real literature databases. Because of that, the easiest way to try it is with the demo papers.

## Simple test run

Open a terminal in the project folder and use these commands.

### 1. Start a workflow

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . start --question "Compare process routes for a target material"
```

If you want to choose the folder name for this run:

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . --project-name my_project start --question "Compare process routes for a target material"
```

If you prefer, you can also fill in a text file and start from that:

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . start-from-file --input-file templates/user_input_template.txt
```

The input template is here:

- `templates/user_input_template.txt`

### 2. Load demo papers

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . load-demo-papers
```

### 3. Run the workflow

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . run
```

At this point, the workflow should pause for shortlist approval.

### 4. Check the status in simple words

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . explain-status
```

You can also inspect what the workflow found:

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . show-search-results
PYTHONPATH=src python3 -m literature_review_agent --project-root . show-shortlist
```

### 5. Approve the shortlist

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . approve-shortlist --all
```

### 6. Run again to finish

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . run
```

### 7. Read the result

```bash
PYTHONPATH=src python3 -m literature_review_agent --project-root . show-recommendations
PYTHONPATH=src python3 -m literature_review_agent --project-root . show-final-brief
```

## What files matter most

- `projects/<project_name>/state/workflow_state.yml`
  This is the master progress file.
- `projects/<project_name>/state/shortlist.yml`
  This is where shortlist approval is tracked.
- `projects/<project_name>/state/recommendations.yml`
  This stores the final route recommendations.
- `projects/<project_name>/outputs/final_brief.md`
  This is the final brief generated by the scaffold.
- `projects/<project_name>/outputs/search_results.txt`
  This is a plain-English search summary.
- `projects/<project_name>/outputs/shortlist_review.txt`
  This is a plain-English shortlist review file.
- `projects/<project_name>/outputs/recommendations.txt`
  This is a plain-English recommendation summary.

## What is still missing

The current version still needs:

- real literature search,
- real paper reading and extraction,
- stronger route comparison logic,

For now, this version is mainly a working skeleton that shows how the workflow behaves.
